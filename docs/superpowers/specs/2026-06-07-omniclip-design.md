## 1. 产品定位

**OmniClip** — 让复制不只是暂存。集成了 AI 语义召回与多模态即时处理的下一代智能剪贴板管理器。

### 1.1 核心痛点

- 复制的内容经常被新复制覆盖，无法找回
- 想找前面复制过的内容，只能靠记忆
- 没有对剪贴板历史的总结和管理能力
- 复制的东西就是"临时搬运"，用完即弃

### 1.2 核心价值

| 价值 | 描述 |
|------|------|
| **不丢失** | 每一次 Ctrl+C 都被忠实地记录下来 |
| **可检索** | 语义搜索 + 关键词搜索 + 时间线 + 标签筛选，四种方式找回 |
| **可理解** | AI 自动分类、智能总结、知识提取 |
| **隐私优先** | 数据全本地存储，AI 优先本地推理，云端 API 由用户决定 |

### 1.3 目标用户

开发者、知识工作者、设计师、任何频繁使用复制粘贴的 Windows 用户。

## 2. 技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| **平台** | Windows 优先，后续跨平台 | 目标用户基数和开发成本平衡 |
| **语言与框架** | C# .NET 8 + WPF | Windows 原生体验最佳，系统级 API 支持完整，AI 生态成熟 |
| **本地 AI 推理** | ONNX Runtime + all-MiniLM-L6-v2 (multilingual) | 语义嵌入和分类全离线，384维向量，~80MB 模型 |
| **数据库** | SQLite (Microsoft.Data.Sqlite) | 轻量零配置，足够个人使用场景 |
| **OCR** | Windows.Media.Ocr | 系统内置，离线，中文支持好 |
| **文档解析** | Open XML SDK (Word/Excel), PdfPig (PDF) | .NET 生态成熟库 |
| **云端 LLM** | DeepSeek / OpenAI / Claude (用户配置 Key) | 用户完全控制，API Key 本地加密存储 |
| **打包分发** | MSIX / Inno Setup | Windows 标准安装方式 |

## 5. 数据模型

### 5.1 主表: clipboard_entries

```sql
CREATE TABLE clipboard_entries (
    id              TEXT PRIMARY KEY,          -- UUID
    content_type    TEXT NOT NULL,             -- text | image | file | url | code
    plain_text      TEXT,                      -- 提取后的纯文本（全文搜索源）
    rich_text       TEXT,                      -- 富文本/HTML
    file_path       TEXT,                      -- 大文件本地路径
    file_name       TEXT,                      -- 原始文件名
    source_app      TEXT,                      -- 来源应用名
    source_window   TEXT,                      -- 来源窗口标题
    content_hash    TEXT NOT NULL,             -- SHA256
    char_count      INTEGER,                   -- 字符数
    tags            TEXT,                      -- JSON 数组
    is_favorite     INTEGER DEFAULT 0,
    is_pinned       INTEGER DEFAULT 0,
    is_sensitive    INTEGER DEFAULT 0,
    in_knowledge_base INTEGER DEFAULT 0,       -- 是否已加入知识库
    importance_score REAL DEFAULT 0,           -- 重要性评分
    created_at      TEXT NOT NULL,             -- ISO 8601
    last_accessed   TEXT
);

CREATE INDEX idx_entries_created ON clipboard_entries(created_at DESC);
CREATE INDEX idx_entries_type ON clipboard_entries(content_type);
CREATE INDEX idx_entries_tags ON clipboard_entries(tags);
CREATE INDEX idx_entries_kb ON clipboard_entries(in_knowledge_base);
```

### 5.2 向量表: vectors

```sql
CREATE TABLE vectors (
    entry_id    TEXT PRIMARY KEY,
    vector      BLOB NOT NULL,    -- 384 × 4 bytes float32
    model_name  TEXT NOT NULL,
    FOREIGN KEY (entry_id) REFERENCES clipboard_entries(id) ON DELETE CASCADE
);
```

### 5.3 配置表: config

```sql
CREATE TABLE config (
    key     TEXT PRIMARY KEY,
    value   TEXT
);
```

### 5.4 文件存储结构

```
{用户自定义路径}\OmniClip\
├── clipboard.db          -- 元数据 + 文本
├── vectors.db            -- 向量索引
├── knowledge_base.db     -- 知识库（加密）
└── files\                -- 大文件
    └── {yyyy-MM}\
        ├── {uuid}.png
        ├── {uuid}.docx
        └── ...
```

### 5.5 存储估算

| 规模 | 数据库 | 文件 | 总计 |
|------|--------|------|------|
| 1,000 条 | ~2MB | ~50MB | ~52MB |
| 10,000 条 | ~25MB | ~200MB | ~225MB |
| 100,000 条 | ~250MB | ~1GB | ~1.25GB |

默认上限 500MB，用户可调整。超出上限按时间清理非收藏内容。

## 6. AI 管线

### 6.1 内容处理管道

```
Ctrl+C 复制
  │
  ▼
[捕获] Win32 Clipboard Hook → 获取原始数据
  │
  ▼
[识别] 检测类型：文本/图片/文件/URL/代码
  │
  ▼
[提取] 统一产出纯文本
  │    ├── 纯文本 → 直接使用
  │    ├── 富文本/HTML → 提取纯文本
  │    ├── 图片 → Windows.Media.Ocr 提取文字
  │    ├── Word → Open XML SDK 提取文字
  │    ├── PDF → PdfPig 提取文字
  │    ├── Excel → Open XML SDK 提取文字
  │    └── URL → 尝试获取页面标题
  │
  ▼
[向量化] ONNX Runtime 推理 → 384维 float32 向量
  │    延迟: <10ms (短文本) / <100ms (长文本)
  │
  ▼
[存储] SQLite → 元数据 + 文本 + 向量
  │
  ▼
[分类] 规则引擎 + 嵌入匹配 → 自动打标签
  │
  ▼
[入库完成] 可被搜索/浏览
```

### 6.2 语义搜索流程

```
用户输入自然语言搜索词
  │
  ▼
嵌入推理 → 搜索词转为384维向量 (~5ms)
  │
  ▼
暴力搜索所有向量 → 计算余弦相似度
  │    <10,000条 → 全量搜索 (~50ms)
  │    ≥10,000条 → 先按时间粗筛(最近30天)，再搜索
  │
  ▼
混合排序: 语义相似度(70%) + 时间衰减(20%) + 类型匹配(10%)
  │
  ▼
返回 Top 20 结果，显示相似度百分比 + 预览片段
```

### 6.3 自动分类引擎

```
两条路径并行:

① 规则引擎（快速，覆盖 ~80%）
  ├── URL 正则匹配           → 链接
  ├── 代码特征检测            → 代码
  ├── 密码/密钥模式匹配       → 密钥
  ├── 文件路径模式            → 文件
  └── 邮件/手机号模式         → 联系信息

② 嵌入引擎（慢但准，覆盖剩余 ~20%）
  └── 内容向量 vs 预设标签向量 → 相似度 > 阈值 → 匹配对应标签
```

### 6.4 云端 API 集成（用户配置）

```
用户需要配置:
  - API Provider (OpenAI / DeepSeek / Claude / 自定义)
  - API Key (本地 AES 加密存储)
  - API Base URL (自定义时必填)
  - Model 名称

触发云端 API 的场景（每次需用户确认）:
  - 智能总结: 汇总多条剪贴板内容
  - 知识问答: RAG 模式对历史提问
  - 图片理解: 描述图片视觉内容
  - 长文本摘要: 总结单篇长文章

上传前防护:
  1. 本地敏感内容扫描 → 脱敏处理
  2. 弹窗显示将要发送的内容摘要
  3. 用户手动点击"发送"确认
  4. 记录每次 API 调用日志（时间/条数/Token）
```

### 6.5 知识库

```
自动识别重要内容:
  本地模型对每条内容计算"重要性评分":
  ├── 密码/Token/Key 模式      → 高分
  ├── 账号信息组合              → 高分
  ├── 个人信息（地址/电话/卡号） → 高分
  ├── 高频复用的代码片段        → 中高分
  └── 其他普通文本              → 低分

重要性评分 > 阈值:
  → 提示用户"是否添加到知识库？"
  → 可设置自动加入（开机自启时生效）

知识库特征:
  - 单独加密存储 (knowledge_base.db)
  - 不随历史清理策略删除
  - 单独的知识库视图浏览
  - 支持手动添加/编辑/删除
```

## 7. 存储策略

### 7.1 存储路径

- 默认: `%APPDATA%\OmniClip\data\`
- 用户可在设置中自定义任意路径

### 7.2 清理策略

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 保留天数 | 30天 | 超期自动删除 |
| 最大空间 | 500MB | 超出则清理最旧的非收藏内容 |
| 图片/文件上限 | 50MB/条 | 超大文件只存引用 |
| 收藏内容 | 永久保留 | 不清除 |

### 7.3 排除规则

- 隐私应用白名单（从指定程序复制的内容不记录）
- 内容长度下限（短于 N 字符不记录）
- 正则黑名单（匹配特定模式不记录）

## 9. 安全与隐私

### 9.1 数据安全

- 所有剪贴板数据**仅存本地**，不上传任何服务器
- 知识库数据单独 AES 加密存储
- API Key 使用 Windows DPAPI 加密存储（绑定当前用户账户）

### 9.2 隐私保护

- 可配置"隐私应用"列表（密码管理器等），从其复制的信息不记录
- 敏感内容本地识别（密码模式、银行卡号、身份证号等正则 + 嵌入模型）
- 云端 API 调用前必须用户手动确认
- 上传前本地脱敏处理

### 9.3 用户控制

- 随时清空全部历史
- 单条删除
- 暂停/恢复监听
- 导出数据前脱敏选项

## 11. 非功能需求

### 11.1 性能

| 指标 | 目标 |
|------|------|
| 复制到可检索延迟（文本） | < 200ms |
| 复制到可检索延迟（图片含OCR） | < 1s |
| 嵌入推理延迟（短文本） | < 10ms |
| 语义搜索（1万条） | < 100ms |
| 启动时间 | < 2s |
| 后台空闲内存 | < 100MB |
